1.- Comandos, módulos y directivas de Apache Server.

Comandos

  • apt install apache2 (el -get se quita a partir de la v18.04 de Ubuntu).
    • Comando usado para instalar Apache.
  • apache2ctl configtest
    • Comando usado para checkear un archivo de Apache.
  • a2ensite dominio.com
    • Comando usado para activar un dominio si este dominio ha sido creado anteriormente.
  • a2dissite dominio.com.conf
    • Comando usado para eliminar un dominio activo.
  • /etc/init.d/apache2 reload o service reload.
    • Comando usado para recargar sin reiniciar el servicio de Apache.
  • /etc/init.d/apache2 restart o service restart.
    • Comando usado para reiniciar el servicio de Apache.
  • /etc/init.d/apache2 stop o service stop.
    • Comando usado para parar el servicio de Apache.
  • /etc/init.d/apache2 start o service start.
    • Comando usado para arrancar el servicio de Apache.
  • /etc/init.d/apache2 status o service status.
    • Comando usado para ver si se encuentra activo el servicio de Apache.

Módulos

  • /etc/apache2/sites-available
    • En este archivo se encuentran los módulos disponibles de Apache.
  • /etc/apache2/sites-enabled
    • En este archivo se encuentran los módulos activos de Apache.
  • /etc/apache2/apache2.conf
    • En este archivo se encuentra la configuración de Apache.

Directivas

  • ServerRoot
    • Esta directiva especifica el directorio de nivel superior que tiene el contenido web
  • IfModule
    • Las etiquetas <IfModule></IfModule> crean un contenedor condicional que sólo es activado si el módulo especificado es cargado.
  • Listen
    • Esta directiva identifica los puertos en los que el sevidor Web aceptará las peticiones entrantes.
  • Include
    • Esta directiva permite que se incluyan otros archivos de configuración en el tiempo de ejecución.
  • LoadModule
    • Esta directiva es usada para cargar en módulos Dynamic Shared Object (DSO).
  • ServerName
    • Es directiva es usada para configurar un nombre de servidor y un número de puerto para el servidor.
  • DocumentRoot
    • Esta directiva se usa para indicar el directorio que contiene la mayoría de los archivos HTML que se entregarán en respuesta a peticiones.
  • Directory
    • Las etiquetas <Directory /path/to/directory></Directory> se usan para crear un contenedor que se utiliza para cercar un grupo de directivas de configuración que sólo se aplican a un directorio y sus subdirectorios específicos.
  • AllowOverride
    • Esta directiva endica si puede o no ignorar cualquiera de las Options por las declaraciones en un archivo .htaccess.
  • Alias
    • Esta directiva hace accesibles a los directorios fuera del directorio indicado en la directiva DocumentRoot.
  • Redirect
    • Cuando se mueve a una página web, se puede utilizar esta directiva para crear asignaciones de la ubicación del archivo a un nuevo URL.
  • ErrorDocument
    • Esta directiva asocia un código de respuesta HTTP con un mensaje o una URL para que sea devuelto al cliente.

Estas son solo algunas de la directivas que hemos usado a lo largo del curso, pero existen más y pueden verse en este enlace.
